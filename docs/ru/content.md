# Создание контента

Xingluo использует Astro Content Collections для управления контентом, поддерживая Markdown (`.md`) и MDX (`.mdx`, требуется `features.mdx`).

## Коллекции контента

Две коллекции определены в [`src/content.config.ts`](../src/content.config.ts):

| Коллекция | Директория           | Назначение                                        |
| --------- | -------------------- | ------------------------------------------------- |
| `posts`   | `src/content/posts/` | Записи блога                                      |
| `pages`   | `src/content/pages/` | Статические страницы (например, страница "О нас") |

Соглашения об именовании файлов:

- Файлы или директории, начинающиеся с `_`, игнорируются (удобно для черновиков)
- Если MDX включён, собираются `**/*.{md,mdx}`; в противном случае только `**/*.md`
- URL-адреса записей выводятся из пути к файлу (см. раздел маршрутизации в [Обзор архитектуры](./architecture.md))

## Frontmatter записи

Полные поля для коллекции `posts`:

```markdown
---
title: "Заголовок записи" # обязательно
pubDatetime: 2026-06-19T10:00:00+08:00 # обязательно, время публикации
modDatetime: 2026-06-20T10:00:00+08:00 # опционально, время обновления
description: "Краткое описание, используется для SEO и списков" # обязательно
tags: ["Astro", "blog"] # опционально, по умолчанию ["others"]
featured: true # опционально, избранное (показывается на главной)
draft: false # опционально, черновики не публикуются
author: "Xingluo" # опционально, по умолчанию site.author
ogImage: "./cover.png" # опционально, OG-изображение (импорт изображения или строковый путь)
heroImage: "./hero.png" # опционально, изображение-герой (показывается между кнопкой назад и заголовком, также справа в карточках)
heroImageFit: "cover" # опционально, режим подгонки изображения-героя (cover обрезка для заполнения / contain полное масштабирование), по умолчанию cover
canonicalURL: "https://..." # опционально, каноническая ссылка
hideEditPost: false # опционально, скрыть ссылку на редактирование
timezone: "Asia/Shanghai" # опционально, переопределить часовой пояс сайта
---
```

### Справочник полей

| Поле             | Тип                      | По умолчанию    | Примечания                                                                                                                                                                                        |
| ---------------- | ------------------------ | --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `title`          | string                   | обязательно     | Заголовок записи                                                                                                                                                                                  |
| `pubDatetime`    | date                     | обязательно     | Время публикации, ISO 8601                                                                                                                                                                        |
| `modDatetime`    | date                     | —               | Время обновления; показывает метку "обновлено"                                                                                                                                                    |
| `description`    | string                   | обязательно     | Краткое описание, используется в meta, RSS и карточках списка                                                                                                                                     |
| `tags`           | string[]                 | `["others"]`    | Массив тегов; страницы тегов генерируются автоматически                                                                                                                                           |
| `featured`       | boolean                  | —               | Показывается в разделе "Избранное" на главной странице                                                                                                                                            |
| `draft`          | boolean                  | —               | Черновик; фильтруется в производственных сборках (виден в разработке)                                                                                                                             |
| `author`         | string                   | `site.author`   | Имя автора                                                                                                                                                                                        |
| `ogImage`        | image \| string          | —               | OG-изображение; `image()` проходит через конвейер ресурсов Astro, строка — путь в `public/` или внешний URL                                                                                       |
| `heroImage`      | image \| string          | —               | Изображение-герой, показывается на странице деталей между кнопкой назад и заголовком, также справа в карточках (управляется `features.showPostCardHero`/`showPostDetailHero`)                     |
| `heroImageFit`   | `"cover"` \| `"contain"` | `"cover"`       | Режим подгонки изображения-героя: `"cover"` обрезает для заполнения (сохраняет пропорции, может обрезать края); `"contain"` полное масштабирование (сохраняет пропорции, может оставлять пустоты) |
| `canonicalURL`   | string                   | —               | Каноническая ссылка, переопределяет значение по умолчанию (см. [SEO](./seo.md))                                                                                                                   |
| `hideEditPost`   | boolean                  | —               | Скрыть ссылку на редактирование для этой записи                                                                                                                                                   |
| `timezone`       | string                   | `site.timezone` | Переопределить часовой пояс отображения для этой записи                                                                                                                                           |
| `locale`         | string                   | `site.lang`     | Язык, на котором написана запись, например `"en"`, `"ja"`. По умолчанию язык сайта                                                                                                                |
| `translationKey` | string                   | —               | Ключ группы перевода: записи с одинаковым ключом являются переводами друг друга. Записи без ключа независимы                                                                                      |
| `category`       | string                   | —               | Категория записи (одно значение), генерирует страницу `/categories/<slug>/`; не задано — без категории                                                                                            |

### Перевод на уровне контента

Используйте поля frontmatter `locale` и `translationKey` для создания многоязычных версий ваших записей:

1. Разместите запись на языке по умолчанию в `src/content/posts/<slug>.md`
2. Разместите переводы в языковых поддиректориях: `src/content/posts/<locale>/<slug>.md` (например, `en/welcome.md`)
3. Установите `locale` на язык перевода и `translationKey` на то же значение, что и у оригинала

Уровень маршрутизации автоматически разрешает правильный перевод для каждого языка и удаляет дубликаты в списках — одна и та же запись на разных языках показывает только одну карточку на язык. Записи без перевода возвращаются к оригинальному содержимому. См. [Интернационализация](./i18n.md).

### Запланированная публикация

Записи с будущими временными метками фильтруются в производстве с использованием допуска `scheduledPostMargin`: если `pubDatetime` находится в пределах окна допуска (по умолчанию 15 минут) от текущего времени, запись считается опубликованной. В режиме разработки видны все нечерновые записи.

## Frontmatter статических страниц

Коллекция `pages` имеет более простые поля:

```markdown
---
title: "О нас"
description: "О этом сайте" # опционально
ogImage: "default-og.jpg" # опционально, только строка
canonicalURL: "https://..." # опционально
---
```

Страница "О нас" получается через `getEntry("pages", "about")` и требует создания `src/content/pages/about.md`.

## Расширения Markdown

Xingluo поставляется со следующими плагинами remark / rehype (см. `astro.config.ts`):

### Содержание

`remark-toc` автоматически генерирует содержание; `remark-collapse` сворачивает его по умолчанию. Вставьте заполнитель в запись:

```markdown
## Содержание

(Содержание будет заполнено автоматически)
```

### Выноски (Callouts)

`rehype-callouts` поддерживает выноски в стиле Obsidian:

```markdown
> [!NOTE]
> Note content

> [!WARNING]
> Warning content

> [!TIP]
> Tip content
```

Поддерживаемые типы: `NOTE`, `TIP`, `INFO`, `WARNING`, `DANGER`, `SUCCESS`, `QUESTION`, `FAILURE` и другие.

### Подсветка кода

Shiki с двойной темой (светлая `min-light`, тёмная `night-owl`) поддерживает:

- Подсветка строк: ` ```js {1,3-5} `
- Подсветка слов: ` ```js /word/ `
- Маркеры diff: `+` / `-` в начале строки
- Метки имени файла: ` ```js file=src/index.ts ` или `filename=src/index.ts`

```js file=example.js
function hello() {
  console.log("hello"); // подсвеченная строка
}
```

### Tables

Широкие таблицы автоматически оборачиваются в контейнер с горизонтальной прокруткой (плагин `rehypeWrapTable`), предотвращая переполнение на узких экранах.

## Поддержка MDX

Если `features.mdx` включён (по умолчанию), вы можете использовать файлы `.mdx` для компонентно-ориентированного создания контента.

### Пользовательские компоненты

Встроенные MDX-компоненты Xingluo находятся в [`src/components/mdx/`](../src/components/mdx) и импортируются из единой точки входа:

```mdx
import { APlayer, DPlayer } from "@/components/mdx";

# Моя запись

<APlayer
  audio={[
    {
      name: "Песня",
      artist: "Исполнитель",
      url: "/audio.mp3",
      cover: "/cover.jpg",
    },
  ]}
/>

<DPlayer video={{ url: "/video.mp4", pic: "/cover.jpg" }} />
```

Подробнее см. [Медиаплееры](./media-players.md).

### Отключение MDX

При `features.mdx: false`:

- Интеграция `mdx()` не загружается
- Глоб коллекции контента соответствует только `*.md` (существующие файлы `.mdx` не собираются)
- Результат сборки не содержит MDX-среды выполнения

## Комментарии

Система комментариев автоматически отображается в нижней части страниц деталей записей (настройте провайдера в `features.comments`). См. [Система комментариев](./comments.md).

## Время чтения

Расчётное время чтения автоматически отображается на страницах деталей записей и карточках списка:

- **Языки CJK** (zh-cn, ja, ko): рассчитывается по количеству символов CJK, ~400 символов в минуту
- **Другие языки**: рассчитывается по количеству слов (разделение по пробелам), ~200 слов в минуту
- Результат округляется вверх, минимум 1 минута

Перед подсчётом блоки кода, HTML-теги, URL-адреса Markdown-ссылок и другое нетекстовое содержимое удаляются, чтобы оценка была близка к реальному объёму чтения. Никакой настройки не требуется.

## Связанные записи

До 2 связанных записей отображаются в нижней части страниц деталей записей (после навигации предыдущая/следующая):

- Сортировка по количеству общих тегов, по убыванию
- При одинаковом количестве сортировка по дате публикации, по убыванию (предпочтение новым записям)
- Раздел не отображается, если ни одна запись не имеет общих тегов
- Автоматически игнорируется поисковым индексом pagefind

Настройка не требуется.

## Фиксированная боковая панель содержания

Фиксированная боковая панель содержания появляется с правой стороны страниц деталей записей на больших экранах (≥1024px):

- Автоматически генерируется из заголовков h2–h6 статьи, представлена в виде плоского списка с отступами
- Отступы отражают глубину заголовка (h3 имеет на один уровень отступа больше, чем h2)
- Текущий раздел подсвечивается при прокрутке (IntersectionObserver)
- Нажатие на элемент содержания плавно прокручивает к соответствующему заголовку
- Скрыто на маленьких экранах (мобильных), где доступно встроенное сворачиваемое содержание

Генерируется из `headings`, возвращаемых `render()` Astro — автору не нужно вручную поддерживать содержание. Встроенное сворачиваемое содержание `remark-toc` (напишите `## Содержание` в своей записи) сосуществует с боковой панелью для использования на маленьких экранах.

## Категории

Назначьте категорию записи через поле frontmatter `category` (одна строка):

```yaml
---
title: "Моя запись"
category: "tutorial"
---
```

- Страница категории находится по адресу `/categories/<slug>/`; slug нормализуется через `slugifyStr` (CJK сохраняется, латиница в нижнем регистре с дефисами)
- Индекс категорий `/categories/` перечисляет все категории
- Карточки записей и страницы деталей автоматически показывают ссылку на категорию (нажмите, чтобы перейти на страницу категории)
- Запись принадлежит максимум одной категории (в отличие от нескольких `tags`); записи без `category` не отображаются ни в одной категории
- Страницы категорий используют `posts.perPage` для пагинации и поддерживают многоязычные зеркальные маршруты (`/en/categories/...`)
- Отключите категории через `features.showCategories: false` (пункт навигации и страницы удаляются, sitemap фильтруется)
