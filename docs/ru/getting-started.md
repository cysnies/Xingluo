# Начало работы

Это руководство поможет вам запустить Xingluo для локальной разработки и производственных сборок с нуля.

## Требования

| Зависимость | Мин. версия | Примечания                                          |
| ----------- | ----------- | --------------------------------------------------- |
| Node.js     | 22.12.0     | См. `engines.node` в `package.json`                 |
| pnpm        | 10.x        | Менеджер пакетов (проект использует pnpm workspace) |

> Совет: управляйте версиями Node с помощью [fnm](https://github.com/Schniz/fnm) или [nvm](https://github.com/nvm-sh/nvm).

## Установка

После клонирования репозитория установите зависимости:

```bash
pnpm install
```

После установки зависимостей справочные проекты в `references/` автоматически исключаются из компиляции TypeScript и сборок (см. `exclude` в `tsconfig.json`).

## Локальная разработка

Запустите сервер разработки (по умолчанию `http://localhost:4321/`):

```bash
pnpm dev
```

В режиме разработки:

- Черновики и запланированные посты **все видны** (для предпросмотра); они фильтруются только при производственных сборках
- Изменения коллекций контента вызывают горячую перезагрузку
- Клиентское поведение (переключение темы, View Transitions и т.д.) соответствует продакшену

## Синхронизация типов

После изменения схем или типов коллекций контента запустите sync для обновления `.astro/types.d.ts`:

```bash
pnpm sync
```

## Сборка

Производственная сборка состоит из трех шагов (см. скрипт `build` в `package.json`):

```bash
pnpm build
```

1. **`astro check`**: проверка типов TypeScript и шаблонов Astro; любая ошибка прерывает сборку
2. **`astro build`**: статически генерирует весь сайт в `dist/` (включая динамические OG-изображения, RSS, sitemap, robots.txt, assets UI pagefind)
3. **`pagefind --site dist`**: сканирует `dist/` для создания индекса полнотекстового поиска в `dist/pagefind/`

> Примечание: `pagefind` — это бинарный инструмент, установленный как devDependency; дополнительная настройка не требуется.

## Предпросмотр сборки

Просмотрите результат сборки в `dist/` локально:

```bash
pnpm preview
```

## Качество кода

| Команда             | Назначение                                                 |
| ------------------- | ---------------------------------------------------------- |
| `pnpm format`       | Форматирует весь код с Prettier (включая Astro и Tailwind) |
| `pnpm format:check` | Проверяет соблюдение форматирования (используется в CI)    |
| `pnpm lint`         | Проверки ESLint (включая `eslint-plugin-astro`)            |
