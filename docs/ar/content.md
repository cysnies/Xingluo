# إنشاء المحتوى

يستخدم Xingluo Astro Content Collections لإدارة المحتوى، ويدعم Markdown (`.md`) و MDX (`.mdx`، يتطلب `features.mdx`).

## مجموعات المحتوى

تم تعريف مجموعتين في [`src/content.config.ts`](../src/content.config.ts):

| المجموعة | الدليل               | الغرض                          |
| -------- | -------------------- | ------------------------------ |
| `posts`  | `src/content/posts/` | مقالات المدونة                 |
| `pages`  | `src/content/pages/` | الصفحات الثابتة (مثل صفحة حول) |

اتفاقيات تسمية الملفات:

- الملفات أو الدلائل التي تبدأ بـ `_` يتم تجاهلها (مفيد للمسودات)
- عند تمكين MDX، يتم جمع `**/*.{md,mdx}`؛ وإلا فقط `**/*.md`
- عناوين URL للمقالات تُشتق من مسار الملف (انظر قسم التوجيه في [نظرة عامة على الهندسة](./architecture.md))

## Frontmatter المقالات

الحقول الكاملة لمجموعة `posts`:

```markdown
---
title: "عنوان المقال" # مطلوب
pubDatetime: 2026-06-19T10:00:00+08:00 # مطلوب، وقت النشر
modDatetime: 2026-06-20T10:00:00+08:00 # اختياري، وقت التحديث
description: "ملخص، يستخدم لتحسين محركات البحث والقوائم" # مطلوب
tags: ["Astro", "blog"] # اختياري، الافتراضي ["others"]
featured: true # اختياري، مميز (يظهر في الصفحة الرئيسية)
draft: false # اختياري، المسودات لا تُنشر
author: "Xingluo" # اختياري، الافتراضي site.author
ogImage: "./cover.png" # اختياري، صورة OG (استيراد صورة أو مسار نصي)
heroImage: "./hero.png" # اختياري، صورة البطل (تظهر بين زر الرجوع والعنوان، وأيضًا على يمين البطاقات)
heroImageFit: "cover" # اختياري، وضع تكيف صورة البطل (cover قص للتعبئة / contain تحجيم كامل)، الافتراضي cover
canonicalURL: "https://..." # اختياري، الرابط الأساسي
hideEditPost: false # اختياري، إخفاء رابط التعديل
timezone: "Asia/Shanghai" # اختياري، تجاوز المنطقة الزمنية للموقع
---
```

### مرجع الحقول

| الحقل            | النوع                    | القيمة الافتراضية | الملاحظات                                                                                                                                              |
| ---------------- | ------------------------ | ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `title`          | string                   | مطلوب             | عنوان المقال                                                                                                                                           |
| `pubDatetime`    | date                     | مطلوب             | وقت النشر، ISO 8601                                                                                                                                    |
| `modDatetime`    | date                     | —                 | وقت التحديث؛ يظهر علامة "مُحدّث"                                                                                                                       |
| `description`    | string                   | مطلوب             | الملخص، يستخدم في meta و RSS وبطاقات القوائم                                                                                                           |
| `tags`           | string[]                 | `["others"]`      | مصفوفة الوسوم؛ صفحات الوسوم تُنشأ تلقائيًا                                                                                                             |
| `featured`       | boolean                  | —                 | يظهر في قسم "مميز" في الصفحة الرئيسية                                                                                                                  |
| `draft`          | boolean                  | —                 | مسودة؛ تُصفى في بناءات الإنتاج (مرئية في التطوير)                                                                                                      |
| `author`         | string                   | `site.author`     | اسم المؤلف                                                                                                                                             |
| `ogImage`        | image \| string          | —                 | صورة OG؛ `image()` تمر عبر خط أنابيب Astro، النص هو مسار `public/` أو رابط خارجي                                                                       |
| `heroImage`      | image \| string          | —                 | صورة البطل، تظهر في صفحة التفاصيل بين زر الرجوع والعنوان، وأيضًا على يمين بطاقات المقالات (تتحكم بها `features.showPostCardHero`/`showPostDetailHero`) |
| `heroImageFit`   | `"cover"` \| `"contain"` | `"cover"`         | وضع تكيف صورة البطل: `"cover"` قص للتعبئة (يحافظ على نسبة الأبعاد، قد يقص الحواف)؛ `"contain"` تحجيم كامل (يحافظ على نسبة الأبعاد، قد يترك فراغات)     |
| `canonicalURL`   | string                   | —                 | الرابط الأساسي، يتجاوز الافتراضي (انظر [تحسين محركات البحث](./seo.md))                                                                                 |
| `hideEditPost`   | boolean                  | —                 | إخفاء رابط التعديل لهذا المقال                                                                                                                         |
| `timezone`       | string                   | `site.timezone`   | تجاوز المنطقة الزمنية المعروضة لهذا المقال                                                                                                             |
| `locale`         | string                   | `site.lang`       | اللغة التي كُتب بها المقال، مثلاً `"en"`، `"ja"`. الافتراضي لغة الموقع عندما لا يُحدد                                                                  |
| `translationKey` | string                   | —                 | مفتاح مجموعة الترجمة: المقالات التي تشارك نفس المفتاح هي ترجمات لبعضها البعض. المقالات بدون مفتاح مستقلة                                               |
| `category`       | string                   | —                 | تصنيف المقال (قيمة مفردة)، يُنشئ صفحة `/categories/<slug>/`؛ عدم التحديد يعني عدم وجود تصنيف                                                           |

### الترجمة على مستوى المحتوى

استخدم حقول frontmatter `locale` و `translationKey` لإنشاء نسخ متعددة اللغات من مقالاتك:

1. ضع المقال باللغة الافتراضية في `src/content/posts/<slug>.md`
2. ضع الترجمات في الدلائل الفرعية للغة: `src/content/posts/<locale>/<slug>.md` (مثل `en/welcome.md`)
3. عيّن `locale` إلى لغة الترجمة و `translationKey` إلى نفس القيمة الأصلية

طبقة التوجيه تحل تلقائيًا الترجمة الصحيحة لكل لغة وتزيل التكرار في القوائم — نفس المقال بلغات مختلفة يُظهر بطاقة واحدة فقط لكل لغة. المقالات بدون ترجمة ترجع إلى المحتوى الأصلي. انظر [التدويل](./i18n.md).

### النشر المجدول

المقالات ذات الطوابع الزمنية المستقبلية تُصفى في الإنتاج باستخدام تفاوت `scheduledPostMargin`: إذا كان `pubDatetime` ضمن نافذة التفاوت (الافتراضي 15 دقيقة) من الوقت الحالي، يُعتبر المقال منشورًا. في التطوير، جميع المقالات غير المسودة مرئية.

## Frontmatter الصفحات الثابتة

مجموعة `pages` لها حقول أبسط:

```markdown
---
title: "حول"
description: "حول هذا الموقع" # اختياري
ogImage: "default-og.jpg" # اختياري، نص فقط
canonicalURL: "https://..." # اختياري
---
```

يتم جلب صفحة "حول" عبر `getEntry("pages", "about")` وتتطلب إنشاء `src/content/pages/about.md`.

## تحسينات Markdown

يأتي Xingluo مع إضافات remark / rehype التالية (انظر `astro.config.ts`):

### جدول المحتويات

`remark-toc` يُنشئ جدول المحتويات تلقائيًا؛ `remark-collapse` يطويه افتراضيًا. أدخل العنصر النائب في المقال:

```markdown
## جدول المحتويات

(يتم ملء جدول المحتويات تلقائيًا هنا)
```

### التنبيهات

`rehype-callouts` يدعم تنبيهات نمط Obsidian:

```markdown
> [!NOTE]
> محتوى الملاحظة

> [!WARNING]
> محتوى التحذير

> [!TIP]
> محتوى النصيحة
```

الأنواع المدعومة: `NOTE`, `TIP`, `INFO`, `WARNING`, `DANGER`, `SUCCESS`, `QUESTION`, `FAILURE` والمزيد.

### تظليل الكود

Shiki ذو السمة المزدوجة (فاتح `min-light`، داكن `night-owl`) يدعم:

- تظليل السطر: ` ```js {1,3-5} `
- تظليل الكلمة: ` ```js /word/ `
- علامات الفرق: `+` / `-` في بداية السطر
- تسميات اسم الملف: ` ```js file=src/index.ts ` أو `filename=src/index.ts`

```js file=example.js
function hello() {
  console.log("hello"); // سطر مظلل
}
```

### الجداول

الجداول العريضة تُلف تلقائياً في حاوية قابلة للتمرير أفقيًا (إضافة `rehypeWrapTable`)، مما يمنع الفائض على الشاشات الضيقة.

## دعم MDX

عند تمكين `features.mdx` (افتراضي)، يمكنك استخدام ملفات `.mdx` للتأليف القائم على المكونات.

### المكونات المخصصة

مكونات MDX المضمنة في Xingluo موجودة في [`src/components/mdx/`](../src/components/mdx) ويتم استيرادها من مدخل موحد:

```mdx
import { APlayer, DPlayer } from "@/components/mdx";

# مقالتي

<APlayer
  audio={[
    { name: "أغنية", artist: "فنان", url: "/audio.mp3", cover: "/cover.jpg" },
  ]}
/>

<DPlayer video={{ url: "/video.mp4", pic: "/cover.jpg" }} />
```

انظر [مشغلات الوسائط](./media-players.md) للتفاصيل.

### تعطيل MDX

عند `features.mdx: false`:

- لا يتم تحميل تكامل `mdx()`
- نمط glob لمجموعة المحتوى يطابق فقط `*.md` (ملفات `.mdx` الموجودة لا تُجمع)
- مخرجات البناء لا تحتوي على MDX runtime

## التعليقات

يتم عرض نظام التعليقات تلقائياً في أسفل صفحات تفاصيل المقالات (قم بتكوين المزود في `features.comments`). انظر [نظام التعليقات](./comments.md).

## وقت القراءة

يتم عرض وقت القراءة المقدر تلقائيًا على صفحات تفاصيل المقالات وبطاقات القوائم:

- **لغات CJK** (zh-cn، ja، ko): تُحسب بعدد أحرف CJK، ~400 حرف في الدقيقة
- **اللغات الأخرى**: تُحسب بعدد الكلمات (مفصولة بمسافات)، ~200 كلمة في الدقيقة
- النتيجة تُقرّب لأعلى، حد أدنى دقيقة واحدة

قبل العد، يتم إزالة كتل التعليمات البرمجية ووسوم HTML وروابط Markdown والمحتوى غير النصي الآخر للحفاظ على التقدير قريباً من حجم القراءة الفعلي. لا يلزم أي تكوين.

## المقالات ذات الصلة

يتم عرض ما يصل إلى مقالتين ذات صلة في أسفل صفحات تفاصيل المقالات (بعد التنقل السابق/التالي):

- مرتبة بعدد الوسوم المشتركة، تنازليًا
- نفس الدرجة تُرتّب حسب تاريخ النشر، تنازليًا (تفضيل المقالات الأحدث)
- لا يتم عرض القسم عندما لا تشارك أي مقالات الوسوم
- يتم تجاهلها تلقائيًا بواسطة فهرس بحث pagefind

لا يلزم أي تكوين.

## الشريط الجانبي الثابت لجدول المحتويات

يظهر شريط جانبي ثابت لجدول المحتويات على الجانب الأيمن من صفحات تفاصيل المقالات على الشاشات الكبيرة (≥1024 بكسل):

- يتم إنشاؤها تلقائيًا من العناوين h2–h6 في المقالة، معروضة كقائمة مسطحة ذات مسافة بادئة
- تعكس المسافة البادئة عمق العنوان (h3 لديه مستوى واحد إضافي من المسافة البادئة مقارنة بـ h2)
- يتم تمييز القسم الحالي أثناء التمرير (IntersectionObserver)
- النقر على إدخال جدول المحتويات يؤدي إلى التمرير السلس إلى العنوان المقابل
- مخفي على الشاشات الصغيرة (الجوال)، حيث يتوفر جدول المحتويات القابل للطي المضمن

يتم إنشاؤها من `headings` التي يعيدها `render()` في Astro — لا يحتاج المؤلف إلى صيانة يدوية لجدول المحتويات. يتعايش جدول المحتويات القابل للطي المضمن `remark-toc` (اكتب `## جدول المحتويات` في مقالتك) مع الشريط الجانبي للاستخدام على الشاشات الصغيرة.

## التصنيفات

قم بتعيين تصنيف لمقالة عبر حقل frontmatter `category` (سلسلة نصية واحدة):

```yaml
---
title: "مقالتي"
category: "دليل"
---
```

- صفحة التصنيف موجودة على `/categories/<slug>/`؛ يتم تطبيع slug عبر `slugifyStr` (يتم الحفاظ على CJK، اللاتينية بأحرف صغيرة مع واصلات)
- فهرس التصنيفات في `/categories/` يسرد جميع التصنيفات
- بطاقات المقالات وصفحات التفاصيل تظهر تلقائيًا رابط التصنيف (انقر للانتقال إلى صفحة التصنيف)
- تنتمي المقالة إلى تصنيف واحد على الأكثر (على عكس `tags` المتعددة)؛ المقالات بدون `category` لا تظهر في أي تصنيف
- صفحات التصنيف تعيد استخدام `posts.perPage` للترقيم وتدعم المسارات المتطابقة متعددة اللغات (`/en/categories/...`)
- قم بتعطيل التصنيفات عبر `features.showCategories: false` (تتم إزالة إدخال التنقل والصفحات، ويتم تصفية sitemap)
